# react-native-fx engineering guides

These guides define how you write code and docs for react-native-fx. Read the relevant guide before you contribute.

- [Writing Style Guide](./Writing%20Style%20Guide.md) — voice, tone, and formatting for all prose: research notes, READMEs, and TSDoc comments.
- [Swift Style Guide](./Swift%20Style%20Guide.md) — formatting and conventions for the native iOS sources (Swift and Metal).

For the reasoning behind the architecture, read the source of truth in [research](../research). For the non-negotiable working rules, read [CLAUDE.md](../CLAUDE.md).

These guides mirror the conventions of the Expo monorepo, which react-native-fx builds on. Where a guide is silent, defer to the [Expo Documentation Writing Style Guide](https://github.com/expo/expo/blob/main/guides/Expo%20Documentation%20Writing%20Style%20Guide.md) and the [Google developer documentation style guide](https://developers.google.com/style).
