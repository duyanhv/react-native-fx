# U8-003 · the Android interactive-uniform probe fix — review

Date: 2026-06-13
Branch: `integration/0.1.x`. Scope: the one-file rendering-path fix + the `structure.android.md`
pin vs the task contract (`tasks/U8-003/README.md`); evidence at `evidence/device.md` +
`api33-crash-logcat.txt` / `api33-mount-fix-logcat.txt`. The planner thread reviewed and wrote
this; the device run + the API-33 ratification were the maintainer's.

## Verdict

**Approved.** `FxSurfaceShaderView.probeInteractiveUniforms` — which tested interactive-uniform
existence by calling `setFloatUniform` for an absent uniform inside a try/catch on AGSL's
`IllegalArgumentException`, and on API 33 walked into a `RuntimeShader` native error path that
corrupts its own message string and CheckJNI-aborts the process on mount — is replaced by
`scanInteractiveUniforms`, a source-declaration scan of the in-hand AGSL text
(`\buniform\s+\w+\s+<name>\b`, word-boundary-guarded). The `supportsFloatUniform` extension and
its now-unused import are deleted; `onDraw`, the frame loop, and `setShaderId`'s structure are
untouched. iOS (a fixed `FxUniforms` struct uploaded via `setFragmentBytes`, no name lookup) and
the press FSM are untouched. `reviewed` is satisfied by this document.

## Origin — surfaced by U8-002, root-cause corrected

The crash was found by the U8-002 coexistence harness, the first screen to mount `active`-mode
shader surfaces whose AGSL omits the interactive uniforms (`aurora` declares only
time/resolution/intensity; only `dots` declares `pressDepth`/`touch`). The initial device report
diagnosed a corrupt/uninitialised uniform-name string in our Kotlin — the planner corrected this:
fx passes the clean literal `"pressDepth"`, and the garbage bytes are AGSL's own, appended after
the clean `"unable to find uniform named "` prefix during its exception-message construction. The
API-33 AVD tombstone confirms it byte-for-byte (`… 0x6e 0x61 0x6d 0x65 0x64 0x20 0x20 <0xbb> 0x41
0xeb 0x7f`) and the backtrace pins the path: `nativeUpdateUniforms → ThrowIAEFmt →
jniThrowExceptionFmt → NewStringUTF → CheckJNI → JniAbort → signal 6` — the abort fires *while*
AGSL builds the exception the probe expected to catch, so the catch never runs.

## Headless

Gates green at review: packages `tsc`/build/lint (27 files)/`swift:lint`/58 tests, and
`:react-native-fx:compileDebugKotlin` BUILD SUCCESSFUL (the rewritten file recompiled). The scan
returns the correct flags by reading the fixtures: `aurora` → both false (no interactive writes),
`dots` → both true. The `\b` guard correctly rejects `dots`'s in-body local `touchPoint`.

## Device — the gold-standard proof (repro validated, then fixed)

On an android-33 AVD (`fx_api33`, Android 13; arm64 on an Apple-silicon host — both binding
constraints met, API 33 + a hardware GPU that runs AGSL), the run did the rigorous thing:

1. **Repro validated.** The fix was surgically reverted (`git stash push -- <file>`, old probe
   confirmed back); the coexistence screen then SIGABRT'd on mount with the *exact* physical
   tombstone signature (`api33-crash-logcat.txt`). This proves the AVD is a faithful repro — only
   then does a crash-free mount with the fix mean anything.
2. **Fix proven.** Restored (`git stash pop`, hard-verified present) and rebuilt → mount **5/5
   crash-free** on API 33, pid stable, 0 abort matches (`api33-mount-fix-logcat.txt`).

Plus API-35 regression-safety from the prior run (the rewrite did not break the working path).
The fix was verified restored in the working tree at review (5 scan markers, 0 probe markers,
`git stash` empty, only `FxSurfaceShaderView.kt` modified under `packages/`).

## Residual (accepted by the maintainer)

The `dots`-active *visible* press bulge could not be observed on device: the only `dots` surface
is the U8-002 nested inner, which is occluded by the outer's shader (see `reviews/U8-002.md` /
`30` Decision 6). The write path is proven device-*safe* (the `dots` inner mounts and writes
`pressDepth`/`touch` per frame crash-free) and correct at headless; only the visual confirmation
is blocked. Accepted — not load-bearing for the crash-resolution objective.

## Docs

The no-exception-probe mechanic is pinned in `structure.android.md` §`shader` (why: the API-33
CheckJNI abort; how: scan the source declaration with a word-boundary guard; safety: AGSL strips
only *unused* uniforms, so a declared interactive uniform is always present to write). No ledger
row — a rework of U8-001 code; RT-001 closes at U8-002.
