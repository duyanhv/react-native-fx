## Blockers — the unit cannot run

- **B1 — `FxPressableView` registered in neither module.** `FxModule.swift` (Hosted/Surface/Scroll/Group) and `FxModule.kt` (Hosted/Surface/Group) do not declare `FxPressableView`; JS calls `requireNativeView('ReactNativeFx', 'FxPressableView')` → runtime lookup fails on both platforms.
- **B2 — the recognizer is never activated.** No `pressHandler.update(mode:"active")` exists for `FxPressableView` on either platform (zero matches). iOS never `attach()`es the recognizer; Android `handle()` early-returns while `mode == NONE`. No events, no feedback — content press is always-on and must put the handler into active mode.
- **B3 — iOS effect-surface regression (passive fires `onPressIn`).** `FxSurfaceView.handlePressBegin` dispatches `dispatchShaderPressIn` unconditionally, before the `guard mode == .active`; the original dispatched it only when active. Violates `30` (passive = no `onPress*`). **Android guards this correctly** (`if currentInteractionMode == "active"`) — iOS must match.
- **B4 — Android effect-surface regression (drag-tilt never settles).** The original `handleUp`/`cancel` called `settleDragTilt()` to reset the tilt uniforms on release/cancel; the new `FxSurfaceView.handlePressEnd`/`handlePressCancel` omit it. `dragAxis`/tilt (DEF-011) won't return to neutral after a drag. **iOS preserved this** — Android must restore it.

## Quality / verify (same round)

- **Q1 (iOS feedback)** — `CASpringTimingFunction` on `CAAnimationGroup.timingFunction` is not a real per-property spring; the `asyncAfter(0.4s)` transform reset clobbers a rapid re-press within 400 ms. Prefer `CASpringAnimation` per property with clean cancellation/state tracking.
- **Q2** — Android `handlePressBegin` adds `updateEffectSurfaceVisibility()`, absent from the original `handleDown`; confirm intent (not a surface behavior change).
- **Q3** — passive uniform-not-reset-on-lift (both platforms; the early-return skips the original reset). Minor.
- **Q4** — Android `dispatchTouchEvent` returning `true` blocks the wrapped child's own touches once activated; acceptable for presentational content in V1, but make it deliberate.
