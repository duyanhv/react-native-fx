## Headless round (5069c91, reviewed before the gate)

Clean — scope was exactly the spec's three Kotlin files, no iOS diff, no dependency or
lockfile drift. Design points traced by the reviewer:

- `FxAnimationDriver.createAnimation` seats a `SpringForce()` at construction, so
  `applySpring` cannot hit a null spring.
- The null-spring path explicitly resets stiffness/damping to the stock constants — value-
  identical to the proven U6 default, and necessary: `SpringForce` instances persist across
  envelopes, so without the reset a presence `NO_BOUNCY` would leak into later default
  envelopes. The driver default is unchanged in behavior and self-healing in mechanism.
- All three `animateContentTo` call sites pass `presetSpring()`; the spring family keys on
  the preset while the `motion` map overrides shape only — consistent with the narrowed
  `transition` contract (spring authoring stays deferred).
- Gates re-run by the reviewer: tsc, build, lint, swift:lint, 58 tests,
  `compileDebugKotlin` — all green.

**One review correction (pre-gate):** the spec'd slowed-`animator_duration_scale` capture
was invalid — the package pins `dynamicanimation:1.0.0`, whose physics springs ignore the
global animator scale (the driver gates reduce-motion manually for the same reason). The
runbook was rewritten to per-frame logcat value capture with a default-spring positive
control, following the U6-002 log-and-revert instrumentation pattern.

## Device evidence reconciliation

- **No-overshoot (the MOT-001 cell):** verified by the reviewer against the *full* raw log,
  not just excerpts — across 587 frame samples the only translation excursions outside
  `[0, 176]` sit in the two positive-control windows. Presence envelopes ran
  `stiffness=1500 / dampingRatio=1.0` and approached rest monotonically in both directions.
- **Positive control:** the stock default path (`dampingRatio=0.5`) overshot to −27.3 px
  (~15.5 % of the 176 px travel — matching U7-002's predicted ~16 %), proving the probe
  detects bounce. The "no overshoot" verdict is therefore meaningful, not a broken-probe
  artifact.
- **Retarget regression:** four genuine `retarget=true` events; in every case the cut-short
  `interrupted:true` event strictly precedes the retargeted `animateTo` (1–2 ms), values
  stay continuous, and exactly one `finished=true` completion fires per surviving envelope —
  the U7-001 event-ordering contract holds under the new damping. The run honestly discarded
  its first 120 ms calibration attempts (they landed at the rest boundary) and recalibrated
  to a 20 ms offset for the POCO's ~0.26 s settle.
- **Reduce-motion:** `reduceMotion=true` rows show `animateTo` → `transitionEnd` with zero
  frame samples between; semantic completion still fired.
