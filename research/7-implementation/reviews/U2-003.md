## Review round 1 (fixed and re-verified)

- **Intensity contract contradiction (the substantive finding).** The manifest inherited
  `{ default: 1, range: [0, 3] }` from `02`'s worked example / `data-layer` §1 while the
  shipped, device-verified native clamps to 0...1 with default 0.8 (and `data-layer` §6's
  Metal struct + the material node already said 0..1). Maintainer decided: align to shipped
  reality. Fixed to `{ default: 0.8, range: [0, 1] }` in `manifest.ts`, `02:207`, and
  `data-layer:159`. The conformance suite checks ids, not ranges — value-level drift remains
  convention-guarded; acceptable for V1, noted for the future.
- **Six comments/style violations** — internal-planning refs in shipped code (research-doc
  paths, `(22 decision N)`, `§Events`, `rule #N` — including one inside a manifest `note:`
  data field) and a missing explicit return type on the new `FxSurfaceView` wrapper. All
  fixed; the forbidden-ref sweep over the touched files now returns zero, matching the
  clean baseline the 2026-06-09 audit verified.
- Kotlin broad `Exception` catch in the load proof — accepted as-is (note-only; Kotlin has
  no multi-catch and the handler body is the error event either way).
