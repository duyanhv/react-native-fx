# Review ‚Äî DEF-027: FxReveal anchored-reveal geometric spine (step 1)

Reviewer: planner. Outcome: **approved ‚Äî device-verified both platforms, docs-closed, ready-to-merge.**
Closes **SURF-011**. Step 2 (chrome: radius morph + clip) is a future row.

## What shipped

`FxReveal` ‚Äî a new native lifecycle/geometry-orchestration surface that morphs a collapsed slot
into a preset-owned bottom-half target via the inverse-transform technique (content rasterizes at
full target size, stays sharp), with cross-fade, interruption, and per-phase completion. Public
API: `open` / `collapsed` / `expanded` / `preset="anchoredMorph"` / `transition` / `onTransitionEnd`
/ `style` ‚Äî **no `anchor`/`from` prop, no public `placement` prop** (preset owns the target). The
`target` driver gained non-uniform `scaleX`/`scaleY` + `origin` IR channels (manifest; `41` d16);
public `MotionSpec` keeps uniform `scale`.

## The rounds (recommendation-pass ‚Üí spec ‚Üí build ‚Üí review ‚Üí device ‚Üí rework √ó3)

1. **Spine (step 1).** Non-uniform scale/origin through `FxAnimationVector`/`FxSpring`/both drivers
   (iOS polar read-back replaced with independent-axis decomposition; uniform callers unchanged);
   `FxRevealView`/`FxRevealCoordinator.{swift,kt}`; surface + binding + example + 6 conformance
   tests. Headless review clean; gates green.
2. **Device gate 1.** iOS PASS (rows 1‚Äì7 + 4b). Android FAIL ‚Äî core touch bug: hosted content
   (collapsed card *and* expanded shutter) received no touch. Root cause: `FxRevealView` never
   implemented `ReactPointerEventsView`.
3. **Android touch round 1.** `ReactPointerEventsView`/`AUTO` fixed the collapsed-card tap, but the
   expanded shutter (overflow) stayed dead ‚Äî the in-place-resized expanded layer was a child of a
   still-collapsed-sized host, and RN's `TouchTargetHelper` does not descend for out-of-bounds
   points. Also removed the public `placement` prop here.
4. **Placement-host ratification (maintainer).** An internal fx overlay would reopen DEF-003; a
   visual-only Android panel is not a real interactive spine. Ratified **"app owns the
   bounds-containing placement host; fx owns the reveal animation inside it"** into canon (`54`/`50`).
5. **Reveal-host rework (round 2, cross-platform).** Fill-host + read-collapsed-*slot*-frame on both
   platforms; Android host reverted `AUTO`‚Üí`BOX_NONE` (layers `BOX_NONE`; phase-gated expanded
   interactivity kept); example mounts the reveal in a root `absoluteFill` box-none host. **Device
   gate 2: PASS both** ‚Äî Android shutter takes touch (0‚Üí2) + above the tab bar; iOS regression clean.
6. **Pre-merge tightening.** Removed the JS window-size fallback (expanded wrapper `0√ó0`/`opacity:0`
   until `onLayout` host size); both coordinators defer expanded visibility until
   `hasResolvedContentSize` on the initial `open=true` seat. **W8 PASS both** ‚Äî non-full-window host
   + initial `open=true` shows no wrong-size flash; shutter touchable after layout.
