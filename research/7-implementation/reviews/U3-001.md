## Rules gate (all pass)

- **#1 native owns the frame loop** — fill/material render statically; no per-frame JS, no per-frame
  bridge traffic. Lifecycle pause/resume inherited from `FxNativeView`.
- **#2 agnostic names, platform-native defaults** — props are `effect` (`"fill"`/`"material"`) +
  `intensity`; no `SwiftUI*`/`Compose*` names above the manifest. iOS material defaults to the
  platform's own glass (`.glassEffect`), honoring the law.
- **#3 two substrates** — `FxHostedView` is the decorative, gesture-free `hosted` substrate.
- **#4 never host RN content to sample/distort it** — fill is self-contained generative; iOS material
  is a backdrop blur (samples the compositor backdrop without reparenting RN views, so RN touch
  survives). Docblock states the constraint.
- **#7 Expo Modules + Fabric** — `ExpoView` subclasses, `Module`/`View` DSL, `Prop`/
  `OnViewDidUpdateProps`. No JSI / C++ / HybridObject.
- **#9 reads layout, never writes it** — host fills its own bounds (iOS AutoLayout to self anchors;
  Android `MATCH_PARENT`); does not touch RN layout.

## Design vs contract — reconciled

Initial review found the shipped mechanic diverging from the single-home platform-mechanic docs on both
platforms, with the `proof` claiming reconciliation that had not happened. All resolved:

- **iOS material** now uses `.glassEffect()` on iOS 26+ per `structure.ios.md` (the shipped glass rung),
  with `.ultraThinMaterial`/`.thinMaterial`/`.regularMaterial` selected by intensity below 26. The
  `.opacity(intensity)` flatten — the actual cause of the "faded, opacity-only" device symptom — was
  removed; intensity no longer fades the layer.
- **Android fill** keeps the pragmatic plain-`View` + `onDraw()` + `android.graphics.LinearGradient`
  implementation, and `structure.android.md` now records this as a dated V1 deviation with the Compose
  path (`Brush.linearGradient`) explicitly deferred. The mechanic lives in one place again.
- The contract was **kept intact, not eroded** — `.glassEffect` stayed the iOS rung and was verified at
  the device gate, rather than downgrading the source of truth to match a placeholder.
