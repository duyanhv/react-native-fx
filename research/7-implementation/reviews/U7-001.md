## Device evidence reconciliation (six scenarios × two platforms)

iPhone 17 Pro simulator / iOS 26.5 + POCO F1 (physical) / Android 15 API 35 — all PASS:

- **Deferred unmount** — the child rides the exit and unmounts only on
  `exit finished=true`; Android's frame burst caught the mid-exit retained state directly.
- **Interrupt = retarget** — both interrupt directions, no jump, coherent logs. **The open
  ordering question is answered identically on both platforms:** the cut-short phase emits
  `finished=false interrupted=true` at the interrupt edge, strictly before the retargeted
  phase settles `finished=true`; settled transitions emit exactly one event. Closed in `35`.
- **appear** — `false` is instant with no event; `true` plays the enter with one event.
- **Tappable at rest** — taps register on the settled card (both platforms).
- **Teardown-during-exit** — no crash, no event, clean re-entry (the emit-nothing
  invariant).
- **Reduce-motion** — single-frame placement, events still fire immediately.

Process integrity: the first iOS pass produced a false FAIL caused by a duplicate
"iPhone 17 Pro" simulator — agent-device drove a different sim than the build under
observation. The runner root-caused it with temporary instrumentation (reverted via
`git checkout`), deleted the duplicate, and re-ran clean. The diagnosis is documented in
the evidence and is a harness lesson, not a code finding.
