## Contract reconciliation (all met)

- **Lazy allocation** — `init` builds only the intermediate container; `ensureEffectSurface()`
  (`FxSurfaceView.swift:99`) creates the `MTKView` on the first active shader, called from
  `updateEffectSurfaceVisibility()`. The `pendingShader == ""` guard genuinely moved from gating
  drawing to gating allocation; `metalView` is `MTKView?` and every touch optional-chains.
- **Shared statics are sound** — device/queue/library are instance-invariant; the
  `shaderId`-keyed pipeline cache is cross-instance-valid precisely because the pixel format
  became the fixed `bgra8Unorm` constant. Static-`let` initialization is thread-safe; the
  main-thread-confinement claim on the mutable static `pipelineCache` is accurate (Expo prop
  application and the `MTKView` display-link delegate are both main-thread).
- **No wiring regressions outside the diff** — children still route into
  `intermediateContainer`, so the lazily-added `MTKView` stays frontmost (the old
  `bringSubviewToFront` z-order); interaction lives on `FxSurfaceView` itself and never touches
  the `MTKView`; a nil Metal device falls through without crashing; `tearDownMetal` releases
  only per-view state, the process-lived context by design.
- **Android untouched** — `FxSurfaceView.kt` holds no GPU resources; correct no-op. F11's
  cadence half is correctly split to U2-003.
- **Mechanic pinned** — `structure.ios.md` §Lifecycle records both halves and fixes the stale
  "tear down the pipeline on deinit" wording.
