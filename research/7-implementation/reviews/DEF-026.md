## Finding (caught, fixed, re-verified)

**Index misalignment on unknown effect id (robustness).** `createEffectView` returns `null` for
an unrecognized id; the first cut computed `tilePxHeights`/`tilePositions` over *all* tiles but
collected only non-null views into `tileViews`, so a single dropped tile desynced every following
tile's `opacity`/`scale` mapping (and the intensity-push loop, which iterated `pendingTiles`
against the compacted `tileViews`). Blast radius beyond the offending tile. Well-typed callers
(`ShaderId | 'fill'`) never hit it, but the wire type is `string`.

Fixed in `FxScrollView.kt` only: `tiles.mapNotNull { … }` resolves renderable `(tile, view)` pairs
first; `pxHeights`/`positions`/`mountedTiles`/`tileViews` all derive from that one list (1-to-1 by
construction); the intensity-push branch iterates `mountedTiles`; a dropped id logs `Log.w`. Both
gates re-run green after the fix.
